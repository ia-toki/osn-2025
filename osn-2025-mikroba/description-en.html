Description

Pak Dengklek is in the city of Malang! In Malang, there are many rivers, one of which is the Brantas river. There are many microbes in this river. Pak Dengklek loves playing with microbes and invites you to play together. Before the game starts, you and Pak Dengklek will prepare the following in advance.

  1. First, you will select a positive integer $N$ ($6 \leq N \leq 50$).
  2. Pak Dengklek will then prepare a board of size $N \times N$ where each cell is not sterile
  3. Afterwards, Pak Dengklek will place one microbe onto one of the cells of the board. However, you do not know where Pak Dengklek placed it.

To win the game, you have to guess which cell Pak Dengklek has placed the microbe. You can interact with Pak Dengklek by performing the following operations:

  • QUERY: You will write numbers on the cells of the board as follows:
    • For each not sterile cell, you will write a positive integer between $1$ and $36$ (inclusive).
    • For each sterile cell, you must write $0$.
    • Upon writing on all cells, Pak Dengklek will calculate two scores:
      • Microbe stability score ($S$), defined as the number written on which the microbe is located.
      • The mobility score of the cell where the microbe is located ($M$). The mobility score of a cell $p$ is defined as the number of distinct cells that can be visited from $p$ after performing a series of $4$-direction moves to adjacent cells: up, down, left, right; through cells with the same number written on cell $p$.
    • At the end of this operation, Pak Dengklek will do the following two steps:
      • Pak Dengklek will report to you the two scores $S$ and $M$.
      • Based on the scores $S$ and $M$, Pak Dengklek will change all cells with number other than $S$ or cells with mobility score other than $M$ to sterile. Informally, cells on which the microbes are guaranteed not to exist will become sterile.
  • ANSWER: You guess that the location of the microbe.

Due to time constraints, you can only perform at most three QUERY operation. You will win if you can successfully guess the location of the microbe.

Constraints

  • $6 \leq N \leq 50$

Problem Type Information

This is an "interactive" problem. In this problem, you will interact with the judge's program through standard input (stdin) and standard output (stdout). See the following interaction format carefully.

Interaction

Initially, You will give a value $N$. You will output to the standard output in the following format:

N

Afterwards, You will interact with Pak Dengklek with at most three QUERY operations in the following format:

QUERY
A1,1 A1,2 ... A1,N
A2,1 A2,2 ... A2,N
.  .       .
.   .      .
AN,1 AN,2 ... AN,N

where $A_{i,j}$ represents the value written on the cell on row $i$ and column $j$.

After every QUERY operation, Pak Dengklek will output $S$ and $M$ as described above in the following format:

S M

After all QUERY operations are performed, you have to answer the microbe's location, namely row $X$ and column $Y$, in the following format:

ANSWER
X Y

Immediately after, you program must immediately terminate.

Scoring

There are no subtasks for this problem, but your program will be tested against $50$ test cases instead. For each test case, you will get points if you correctly guess the location of the microbe. Your final points for this problem are the minimum points across all test cases.

For each test case, your points will be graded according to $N$ and the largest number $K$ you wrote across all QUERY operations. The points are calculated as follows:

Condition Score
$6 \lt K \leq 36$ $4$
$4 \lt K \leq 6$ $9$
$K = 4$ $16$
$K \leq 3; 6 \leq N \lt 9$ $26 + 3.0 \times (N - 6)$
$K \leq 3; 9 \leq N \lt 18$ $41 + 2.0 \times (N - 9)$
$K \leq 3; 18 \leq N \lt 30$ $63 + 1.5 \times (N - 18)$
$K \leq 3; 30 \leq N \lt 36$ $84 + 2.5 \times (N - 30)$
$K \leq 3; N \gt 36$ $100$

Your program may receive "Wrong Answer" or "Time Limit Exceeded" (and obtains $0$ points) if it:

  • performs more than three QUERY operations;
  • writes a positive integer on a sterile cell, or $0$ on a non-sterile cell;
  • incorrectly guess the microbe's location during ANSWER operation;
  • produces an output in an incorrect format or beyond the constraints
  • does not perform ANSWER operation.
  • continues to perform other operations after ANSWER operation is performed.

You can view the scoring detail for each test case in the submission page. Specifically:

  • If your program produced an output in the wrong format or beyond the described constraints, then you will receive information regarding the violated constraints.
  • If your program produced an output in the correct format and within the described constraints, but did not correctly answer the microbe's location, you will receive INCORRECT ANSWER with values of $N$, $K$, along with values of $S$ and $M$ for each QUERY operation.
  • If your program correctly guess the microbe's location and within the described constraints, you will receive your score with values of $N$ and $K$.

Note that the grader may be adaptive. In other words, the microbe's original location may change after the interaction starts. However, the responses you received from the interactions so far are still consistent.

On the sample above, $N$ is $6$. In order, the following are operations performed by your program.

  1. After the first QUERY operation, there are eight remaining cells with $S = 3$ written and mobility score $M = 4$.
  2. After the second QUERY operation, there are three remaining cells with $S = 5$ written and mobility score $M = 1$.
  3. After the third QUERY operation, there is only one possible cell where the microbe is located, which is the cell at row $4$ and column $2$.
  4. Finally, the ANSWER operation is performed to answer the cell.

Since your program correctly answer the microbe's location, your program will be awarded some points. Note that ther largest number you ever written across all QUERY operation is $K = 7$. Hence, you will receive $4$ points for this sample.

Warning

Always flush after each output.

    fflush(stdout);
  • For iostream library: 
    std::cout << std::flush;
